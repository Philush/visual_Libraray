## MVP (Phase 1) — Личный инструмент, один пользователь

Цель: рабочее приложение без авторизации, где один пользователь может управлять
своей библиотекой локально или на личном сервере.

### F-01 | Управление книжными шкафами

**Статус:** `[x]`
**Приоритет:** Критический

**Описание:**
Пользователь может создавать, редактировать и удалять книжные шкафы (стеллажи).
Каждый шкаф содержит произвольное количество полок.

**Входные данные:**
- Название шкафа (обязательно)
- Описание (опционально)
- Количество полок при создании (минимум 1)

**Выходные данные:**
- Объект шкафа с вложенным списком полок

**Бизнес-логика:**
1. Шкаф создаётся с указанным количеством полок, пронумерованных сверху вниз
2. Полки можно добавлять и удалять после создания шкафа
3. При удалении полки с книгами — книги не удаляются, а переводятся в "без полки"
4. При удалении шкафа — все книги переводятся в "без полки" (не удаляются)

**Edge cases:**
- Удаление шкафа с книгами → подтверждение + информирование о переносе книг
- Минимум 1 полка в шкафу в любой момент
- Максимальное название: 100 символов

**Ошибки:**
- Пустое название → 422 Unprocessable Entity
- Шкаф не найден → 404 Not Found

### F-02 | Управление книгами (каталог)

**Статус:** `[x]`
**Приоритет:** Критический

**Описание:**
Пользователь может добавлять книги в свою библиотеку, редактировать и удалять их.
Книга существует независимо от полки — она может быть "без полки".

**Входные данные:**
- Название (обязательно)
- Автор (обязательно) — поле с автокомплитом из ранее введённых значений
- ISBN (опционально)
- Количество страниц (опционально) — влияет на ширину корешка при визуализации
- Обложка (опционально) — загрузка файла (JPEG/PNG/WebP, до 5 МБ); хранится в `uploads/covers/`
- Рейтинг (опционально) — 0–5 звёзд, по умолчанию 0
- Цвет корешка (опционально, иначе генерируется автоматически)
- Жанр / теги (опционально) — поле с автокомплитом из ранее введённых значений
- Год издания (опционально)
- Заметки (опционально)

**Выходные данные:**
- Объект книги

**Бизнес-логика:**
1. Книга создаётся без привязки к полке (status: "unplaced")
2. Цвет корешка: если не указан — генерируется детерминированно из названия+автора
   (один и тот же ввод всегда даёт один и тот же цвет)
3. Ширина корешка при визуализации вычисляется из pageCount:
   - < 100 стр → узкий
   - 100–300 стр → средний
   - > 300 стр → широкий
   - Если pageCount не указан → средний

**Edge cases:**
- Дублирование книги (одинаковые title + author) → предупреждение, но не блокировка
- Удаление книги, стоящей на полке → сначала убирается с полки, затем удаляется
- Максимальная длина заметок: 2000 символов

**Ошибки:**
- Пустое название или автор → 422
- Книга не найдена → 404

### F-03 | Привязка книги к полке (Placement)

**Статус:** `[x]`
**Приоритет:** Критический

**Описание:**
Пользователь может разместить книгу на конкретной полке конкретного шкафа.
Книга занимает определённую позицию на полке (порядок слева направо).

**Входные данные:**
- ID книги
- ID полки
- Позиция на полке (порядковый номер)

**Выходные данные:**
- Объект placement (bookId, shelfId, position)

**Бизнес-логика:**
1. Одна книга может стоять только на одной полке одновременно
2. При размещении на новую полку — автоматически убирается с предыдущей
3. Позиции пересчитываются при вставке (сдвиг соседних книг)
4. Книга без полки отображается в общем списке с флагом "unplaced"

**Edge cases:**
- Попытка поставить на несуществующую полку → 404
- Попытка поставить книгу, уже стоящую на этой же полке на ту же позицию → no-op (200)
- Позиция вне диапазона → нормализация (ставится в конец)

**Ошибки:**
- Книга или полка не найдена → 404

### F-04 | Визуализация шкафа

**Статус:** `[x]`
**Приоритет:** Критический

**Описание:**
Пользователь может открыть любой шкаф и увидеть визуальное представление полок
с книгами в виде корешков. Книги отображаются в том порядке, в котором стоят на полке.

**Визуальная логика:**
- Шкаф = прямоугольник с горизонтальными полками
- Полка = горизонтальная плоскость, на которой стоят книги
- Книга = вертикальный прямоугольник (корешок) с названием и автором
- Ширина корешка пропорциональна pageCount
- Цвет корешка — из поля book.spineColor
- Название отображается вертикально вдоль корешка
- Пустая полка отображается как пустое пространство с пунктирной линией
- При наведении на книгу — всплывающая подсказка с полными данными

**Edge cases:**
- Книг на полке больше, чем помещается → горизонтальный скролл внутри полки
- Очень длинное название → truncate с ellipsis
- Нет шкафов → заглушка с CTA "Создать первый шкаф"
- Нет книг на полке → визуальная пустая полка

### F-05 | Drag & Drop: перенос книги на полку

**Статус:** `[x]`
**Приоритет:** Критический

**Описание:**
Пользователь может перетащить книгу из списка книг (или с другой полки) на полку
в визуализации шкафа.

**Входные данные:**
- Источник: книга (из списка или с полки)
- Цель: полка + позиция

**Бизнес-логика:**
1. Drag начинается при захвате книги (mouse down / touch start)
2. Во время drag — визуальный placeholder на целевой полке
3. При drop на полку — оптимистичное обновление UI + API-запрос
4. При ошибке API — откат до предыдущего состояния + уведомление
5. Drag между полками одного шкафа
6. Drag из списка книг на полку
7. Drag с полки обратно в список (убрать книгу с полки)

**Edge cases:**
- Drop вне допустимой зоны → отмена, книга возвращается на место
- Одновременный drag нескольких книг → не поддерживается в MVP
- Touch-устройства → поддерживается через @dnd-kit

### F-06 | Список книг (табличный / карточный вид)

**Статус:** `[x]`
**Приоритет:** Высокий

**Описание:**
Пользователь может просматривать все книги в виде списка с фильтрацией и сортировкой.

**Функции:**
- Поиск по названию и автору
- Фильтр по: автору, жанру, наличию на полке / без полки
- Сортировка по: названию, автору, году, дате добавления
- Переключение вида: список (таблица) / карточки
- Клик на книгу → детальная карточка / модалка

**Edge cases:**
- Пустой результат поиска → заглушка
- Нет книг вообще → CTA "Добавить первую книгу"

### F-07 | Импорт / Экспорт

**Статус:** `[x]`
**Приоритет:** Высокий

**Описание:**
Пользователь может экспортировать список книг в файл и импортировать из файла.

**Форматы:**
- CSV — книги без placement (UTF-8 + BOM для Excel)
- XLSX — книги без placement (стилизованный заголовок, инструкция на втором листе)
- JSON — полный снапшот: книги + placements (для резервного копирования)

**Экспорт:**
- Поля: title, author, isbn, pageCount, genre, publishYear, notes, spineColor
- Имя файла включает дату: library-YYYY-MM-DD.{csv|xlsx|json}
- UI: три кнопки в шапке страницы книг

**Импорт:**
- Обязательные поля: Название, Автор (или title, author в JSON)
- Опциональные: ISBN, Страниц, Жанр, Год издания, Заметки, Цвет корешка
- Формат определяется по расширению файла
- Стратегия дублей: пропустить / обновить — выбирается в UI перед загрузкой
- Toast с итогом: +N создано, N обновлено, N пропущено, N ошибок

**Шаблоны:**
- `docs/templates/import-template.csv` — CSV шаблон с примерами
- `docs/templates/import-template.xlsx` — XLSX шаблон с примерами и листом «Инструкция»

**Статус:** `[x]`

без отдельного модала.

- **Добавить полку:** кнопка «+ Добавить полку» под шкафом

- Кнопка удаления недоступна если полка единственная в шкафу

### UX-04 | Сортировка книг внутри полки

**Статус:** `[x]`

Пользователь может перетащить книгу на другой корешок на той же полке, чтобы
изменить её позицию. Соседние корешки анимированно раздвигаются во время drag.

Реализовано через `@dnd-kit/sortable` (`SortableContext` + `horizontalListSortingStrategy`).
При drop на другую полку — стандартное перемещение (в конец полки).
При drop на конкретный корешок — перемещение на позицию этой книги.

### UX-05 | Ghost при drag совпадает с шириной корешка

**Статус:** `[x]`

DragOverlay при перетаскивании теперь использует реальную ширину корешка (`spineWidth`)
вместо фиксированных 36px. Тонкая книга = тонкий ghost, толстая = широкий.

**Статус:** `[x]`

Пользователь может загрузить изображение обложки (файл с устройства).
- Загрузка через `POST /books/upload-cover` (multer, хранение в `uploads/covers/`)
- В форме добавления/редактирования: превью сразу после выбора файла, кнопка удаления

- На полке: обложка отображается на корешке (object-cover), заменяет цвет + текст
- В модалке деталей книги: обложка `56×80px` в шапке вместо тонкой цветной полоски

### F-08 | Регистрация и авторизация

**Статус:** `[x]`
**Приоритет:** Критический

**Описание:**
Пользователь может создать аккаунт и войти в систему. Сессия поддерживается через JWT-токен (срок действия 30 дней), хранящийся в `localStorage`.

**Входные данные:**
- Регистрация: email (обязательно), пароль (обязательно, min 6 символов), имя (необязательно)
- Вход: email + пароль

**Выходные данные:**
- `{ user: { id, email, name, createdAt, updatedAt }, accessToken: string }`

**Бизнес-логика:**
1. Пароль хэшируется через bcrypt (10 rounds) — plaintext нигде не хранится
2. JWT подписывается секретом `JWT_SECRET` из env, срок действия 30 дней
3. Все защищённые эндпоинты проверяют `Authorization: Bearer <token>`
4. Неавторизованный запрос → 401 Unauthorized
5. Повторная регистрация с тем же email → 409 Conflict
6. Фронтенд: неавторизованный пользователь автоматически перенаправляется на `/login`
7. Авторизованный пользователь, открывающий `/login` или `/register`, перенаправляется на `/library`

**Компоненты:**
- `POST /auth/register` — регистрация
- `POST /auth/login` — вход
- `GET /auth/me` — профиль текущего пользователя (требует токен)
- `AuthContext` (фронтенд) — глобальное состояние авторизации
- `/login`, `/register` — отдельный layout без сайдбара
- `Header` — отображает имя/email пользователя + кнопку «Выйти»
